# Security Policy

## Supported versions

Security fixes are applied to the latest `main` branch state.

## Reporting a vulnerability

Please do **not** open public GitHub issues for suspected security
vulnerabilities.

Instead, report privately to:

- `otilio.latam@gmail.com`

Please include:

- A clear description of the issue.
- Steps to reproduce.
- Potential impact.
- Suggested remediation (if known).

We will acknowledge receipt as quickly as possible and work on a fix before
public disclosure.
